# Product definition

## Positioning

Writer Copilot is an open-source AI writing copilot for knowledge workers who write inside Obsidian.

It is not positioned as a general autonomous agent platform. The user remains in control of the task, context, model, and final write-back decision.

## Core job

Help a knowledge worker turn trusted notes into better writing without leaving the knowledge-base workflow.

## Primary users

- writers and editors;
- researchers and analysts;
- consultants and independent professionals;
- creators who maintain a long-term Obsidian knowledge base.

## Product principles

1. **Writing first** — optimize for discussing, drafting, revising, and continuing text.
2. **Use the user's knowledge** — context comes from the active note and explicitly attached notes.
3. **Human-controlled write-back** — generation never silently changes a note.
4. **Local ownership** — sessions remain local and provider credentials use Obsidian SecretStorage.
5. **Progressive capability** — Agents, Skills, and MCP may extend the product without becoming prerequisites for basic writing.

## 0.1.3 scope

- sidebar chat and local history;
- note, paragraph, and selection context;
- direct model-provider configuration;
- configurable single-turn agent profiles;
- inline writing actions and slash continuation;
- preview, replace, and insert safety controls.

## Out of scope

- vault-wide retrieval or automatic indexing;
- autonomous multi-step execution;
- Skill and MCP runtimes;
- unrestricted file modification;
- mobile support.
